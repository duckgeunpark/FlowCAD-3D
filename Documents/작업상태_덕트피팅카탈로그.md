# 작업 상태 — 덕트 피팅 카탈로그 (standard.pdf 기반)

> 최종 갱신: 2026-06-29 · 기준 브랜치: `main`

## 1. 목표
`standard.pdf`(BNPP HVAC 표준 도면 **0-294-M172-902**)에서 모든 덕트 피팅을 추출해 **선택형 DB**로 구성하고,
사용자가 **피팅을 고르고 치수를 입력하면 3D로 생성**되도록 구현한다. 기존 6종 part_type 대신 표준 카탈로그(28종)가
사용자 노출 단일 소스이며, 각 피팅은 전용(신규) 지오메트리로 표준을 반영한다.

## 3. 지금 동작하는 것 (M2까지)
- 사이드바 상단 **"＋ 표준 피팅 추가 (0-294-M172-902)"** 패널
  1. 카테고리별 28종 피팅 선택 → 2. 치수 입력 → 3. **"모델에 추가"** → 3D 생성·자동 선택
- **UNO 기본값 자동 적용**: `R=W/2`, `R=1.5D`, 고어 수(≤36°:2 / 37–72°:3 / 73–90°:5), `X=75`, `L=1220`
- **렌더 가능**: 직관(사각/원형), 엘보(사각 라디우스·마이터·원형), 변환관(round↔round / rect↔round / rect↔rect), 티 계열
- **pending(선택 가능하나 경고 후 미렌더)**: 오프셋 4종, 와이 2종, 래터럴 2종, 캡 2종, 점검구
  - 백엔드가 `FITTING_PENDING` 경고를 내보내 "표준 형상 구현 중"임을 표시(잘못된 형상 대신 미렌더)

## 4. 추출된 피팅 카탈로그 (28종)
| 카테고리 | 피팅 | 표준 규칙(UNO) |
|---|---|---|
| 직관 | 사각 직관, 원형 직관 | L=1220mm 단부간 |
| 엘보 | 사각 라디우스, 사각 90° 마이터, 원형(고어드) | 사각 R=W/2 · 원형 R=1.5D · 고어 2/3/5 |
| 오프셋 | 사각 직선/라디우스, 원형 마이터/라디우스 | R=W/2 또는 1.5D |
| 변환관 | round→round, rect→round, rect→rect | — |
| 티 | 사각 직각/라디우스, 코니컬, 콤비네이션, 원형 직각, 탭, 45° 탭, 더블 45° 탭 | R=W/2, 넥 NL=200/100 |
| 와이 | 사각 2방향 와이, 대칭 와이 | O=0°~90° |
| 래터럴 | 사각 45°, 코니컬 45° | R=W |
| 캡 | 사각/원형 엔드 캡 | — |
| 부속 | 점검구(Access Door, AD-1~5) | 규격 도어 |

기타 표준 상수: 직선부 `X=75mm`, 점검구 사이즈 표(AD-1~5), 넥 길이 `NL`(급기 200 / 배기·리턴 100), `F`(그릴 20 / 디퓨저 0), `AC=230`(급기).

## 7. 아키텍처 메모
- **단일 소스**: 카탈로그 DB는 shared에 위치, FE(선택·폼)와 BE(지오메트리)가 공유.
- **행 모델**: 피커가 `part_type = 카탈로그 id` + 치수(W/H/D/R/angle/branch…)를 행에 기록 → 기존 assembly 파이프라인(연결 그래프 기반 위치/방향 자동 계산)을 그대로 통과.
- **정규화**: `_normalize_catalog_row`가 W/H/D/to* → size_a/size_b/diameter로 변환해 기존 단면/배치 로직과 호환.
- **증분 전략**: 기하학적으로 동일한 피팅은 검증된 수학을 내부 재사용하되, 신규 형상(offset/wye/lateral/gored/cap)은 M3~M5에서 전용 생성기로 구현.

## 8. 보류 항목 (별도 작업)
- 엘보 절대 방향 `bend_to`: 백엔드 + 테스트 + 테이블 드롭다운 **완료**. 상세 패널 방향 버튼 + "회전 180°씩만" 요구는 **미완**(피팅 카탈로그 작업 후 재개).
- 아이템 상세 패널 **길이 편집기**(`LengthEditor`): 직전 작업에서 추가 완료.

# 작업 상태 — 덕트 피팅 카탈로그 (standard.pdf 기반)

> 최종 갱신: 2026-06-29 · 기준 브랜치: `main`

## 1. 목표
`standard.pdf`(BNPP HVAC 표준 도면 **0-294-M172-902**)에서 모든 덕트 피팅을 추출해 **선택형 DB**로 구성하고,
사용자가 **피팅을 고르고 치수를 입력하면 3D로 생성**되도록 구현한다. 기존 6종 part_type 대신 표준 카탈로그(28종)가
사용자 노출 단일 소스이며, 각 피팅은 전용(신규) 지오메트리로 표준을 반영한다.

## 3. 지금 동작하는 것 (M3까지)
- 사이드바 상단 **"＋ 표준 피팅 추가 (0-294-M172-902)"** 패널
  1. 카테고리별 28종 피팅 선택 → 2. 치수 입력 → 3. **"모델에 추가"** → 3D 생성·자동 선택
- **UNO 기본값 자동 적용**: `R=W/2`, `R=1.5D`, 고어 수(≤36°:2 / 37–72°:3 / 73–90°:5), `X=75`, `L=1220`
- **렌더 가능**: 직관(사각/원형), 엘보(사각 라디우스·마이터·원형 고어드), 오프셋 4종, 변환관(round↔round / rect↔round / rect↔rect), 티 계열
- **pending(선택 가능하나 경고 후 미렌더)**: 와이 2종, 래터럴 2종, 캡 2종, 점검구
  - 백엔드가 `FITTING_PENDING` 경고를 내보내 "표준 형상 구현 중"임을 표시(잘못된 형상 대신 미렌더)

## 7. 아키텍처 메모
- **단일 소스**: 카탈로그 DB는 shared에 위치, FE(선택·폼)와 BE(지오메트리)가 공유.
- **행 모델**: 피커가 `part_type = 카탈로그 id` + 치수(W/H/D/R/angle/branch…)를 행에 기록 → 기존 assembly 파이프라인(연결 그래프 기반 위치/방향 자동 계산)을 그대로 통과.
- **정규화**: `_normalize_catalog_row`가 W/H/D/to* → size_a/size_b/diameter로 변환해 기존 단면/배치 로직과 호환.
- **M3 구현 방식**: round elbow는 `partType=round_elbow` + `gores` scene params로 프론트에서 고어 링 mesh를 생성. offset 4종은 기존 `transition` kind를 재사용하되 `partType`/`offsetStyle`로 전용 offset mesh를 렌더링하고, 출구 포트는 진행 방향을 유지한 채 offset된 위치에 둔다.
- **증분 전략**: 기하학적으로 동일한 피팅은 검증된 수학을 내부 재사용하되, 신규 형상(wye/lateral/cap/access door)은 M4~M5에서 전용 생성기로 구현.

## 8. 보류 항목 (별도 작업)
- 엘보 절대 방향 `bend_to`: 백엔드 + 테스트 + 테이블 드롭다운 **완료**. 상세 패널 방향 버튼 + "회전 180°씩만" 요구는 **미완**(피팅 카탈로그 작업 후 재개).

## 2026-06-29 추가 작업: Excel 정규화 + 3D 표준피팅 추가 재구성

사용자 목표를 "도면의 덕트 내용을 Excel 표로 정리하고, 그 표를 기준으로 전체 덕트 시스템을 3D 생성/검토"하는 방향으로 재정렬했다.

### 완료
- 기존 3D 조인트 quick-add 메뉴(`JointAddMenu`, `addFromJoint`) 제거.
- 상세 패널의 기존 side-tap generic 추가(`addTap`) 제거.
- 3D 조인트 선택은 이제 "선택"만 수행하고, 실제 추가는 표준피팅 패널에서 수행.
- `FittingPicker`가 선택된 열린 조인트 컨텍스트를 읽어 `connect_to_seq`, `connect_port`, `W/H` 또는 `D`를 자동 반영.
- `FittingPicker`에서 elbow는 `bend_to`, offset은 `offset_direction`을 작성.
- Excel/CSV transport 정규화 함수 `normalize_table_rows()` 추가.
- `/api/upload`와 `/api/generate/csv`가 같은 `load_table()` 정규화 경로를 사용.
- 표준피팅 컬럼(`W/H/D/L/R/to*/branch*/offset/X/NL/gores`, `bend_to`, `offset_direction`, `rotation`)을 backend template과 frontend table schema에 추가.
- unknown `part_type`은 더 이상 straight로 조용히 변환되지 않고 명시적 `AssemblyError` 발생.
- 깨진 JSX 문자열이 있던 `DetailPanel`을 ASCII 라벨 기반으로 안정화.
